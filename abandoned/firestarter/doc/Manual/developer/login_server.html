Login Server

Login server requirements doc

  • Multiple login servers, one server can service multiple game servers. Game servers can reference multiple login servers.
  • Allow logins from game servers, league servers, ref servers, bots, and admins.
  • Allow Registration of a user using callsign and password
  • Store usernames and user info in some web accessible database. (pass keys too ? )
  • Allow servers to send in a callsign and passkey and be told if that combo verifies.
  • Send back user info fields for each callsign, such as
    • real name
    • email address
    • permission list (string of named permissions)
    • league
    • team
    • other named fields such as league rank, web URL, and whatnot
  • Allow API for users to change personal info fields ( mail addy, and stuff )
  • Allow user to clear league or team fields but not set them with out permissions
  • Only allow authorized users to set league and team fields.
  • Be able to have a web based or other client based interface for this stuff.
