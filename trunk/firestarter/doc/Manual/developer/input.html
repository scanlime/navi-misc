Input Manager

Input API requirements doc

  • One central input manager
  • Able to handle input from keyboard, mouse (5 buttons + wheel), n joysticks/game pads, and any other device/api in the future.
  • Able to enumerate the devices on a system, and each item on each device
  • Able to register 4 types of "actions", Button, axis, positional, and text.
    • Button actions return a value from 0 to 1 and can have a threshold for Boolean tests. They are for actions such as "fire". They should also be able to return a "toggle" state
    • Axis actions return a value from -1 to 1. They are used for actions such as "turn", They can be bound to mouse, stick, or keys. If they are bound to keys the axis should do some sort of interpolations. They should have features for scale, clamp, and dead zone settings
    • Positional actions return a screen X and Y pos and a click state. These are use for actions such as menu "clicks" and other UI items. they should work with all devices and do similar interpolations.
    • Text actions return the current character or string from the keyboard. These are use for chat or text entry actions.
  • The calling applications can register listeners with each action that will be called each time an action is updated.
  • The calling application can also poll the state of any action at any time.
  • Each action should be able to be assigned to a scope or group so that they can be turned on or off (muted) at any time.
    • Muted actions do not call there listeners and return 0 for all polls
  • Actions can be bound to any item on any device. Actions that take more then one input item must keep items on the same device (keyboard, mouse axes) etc..
  • The input manager should be able to load in a configuration file and bind up a default set of actions.
  • The input manager should allow definition of multiple "binding sets" that can be swapped out at any time.
  • Actions can be rebound to new keys at any time and not change the pointer to the action object.
  • Input manger should be able to poll the state of any item on any device to know "what is down now" for configurations interfaces.
  • The input manager should be able to support future devices by only accessing devices by a base API/class
  • A custom "calibrate" method should be implementing in each device for the weird "devices' like voice commands or some other whackass idea.
