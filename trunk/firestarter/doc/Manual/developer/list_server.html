List Server

List server requirements doc

  • One ceneteral URL for list server access. Must span multiple machines/subnets to allow for redunadncy. All data is shared between all servers.
  • Allow logins from game servers, clients, bots, and admins.
  • collect game server info for publishing.
  • connect back to game servers for verification of server address
  • publish server list to game clients and bots that connect.
  • collect version,OS, and other anonymous info for each connection. Posibly publish those stats to a database
  • Poll or allow a push of game server update data with player counts, game type, and other info
  • Allow any info to be published ( some sort of named field thing )
  • Track current release and dev versions of client and server apps. notify all if they are out of version on connect or on version update
  • Allow baned servers and clients based on hostmask
  • Seperate dev, beta, and release server lists.
  • Allow admin client to login and change local setings, backup database, remove servers, etc..
  • Publish server list database to a some web accessable database (mysql?)
  • More?
