Sound Manager

Sound API requirements doc

  • One central Sound manager
  • Able to load in raw and compressed sounds, wav and ogg at a minimum, mp3 would be nice
  • Able to play 2 types of sounds, effects and songs
  • Effects are played ether as positional or fixed ballance, and are loaded into memory
  • Effects can be looped
  • Positional Effects should use 3d hardware if they can.
  • Positional Effects can have there postion and velocity updated each frame
  • All effects can have there volumes set
  • fixed effects can have there ballance set.
  • both mono and sterio effects are supported
  • All effects are loaded by name when needed, but can be refrenced by an ID for speed
  • Effects that have not been used in N time can be unloaded
  • Songs are background musics that are streamed from the harddrive or a network server
  • Streaming from standard shout/icecast services is optimal but a custom stream can be worked
  • File based songs are supported in a playlist type arangment with next, last, random and repeat features.
  • The FMOD library supports all this allready
