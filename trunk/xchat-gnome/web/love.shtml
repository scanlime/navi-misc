xchat-gnome Love

Feel like giving xchat-gnome some love? There are plenty of tasks that are outside of our core focus at the moment but still ought to be done. If you're giving any of these a try, we're available to answer questions in #xchat-gnome on irc.freenode.net. A lot of these are pretty simple, and we'd love to have the help :)



Art Love
New user status icons

The current bullets next to nicknames in the user list which indicate operator/voice/etc status are inaccessible to colorblind people. In order to fix this, we need new icons that have distinct silhouettes. This art would be useful to the main XChat line as well, since it suffers from the same problem.

Documentation Love
Anything, really

At the moment, we have no user documentation other than the FAQ on this web site. It would be great to have some documentation that can be shown both in yelp and on the web.

Easy Hackery Love
Basic A11y

At the moment, the only a11y is what gtk+ gives us for free. It would be nice if the most important widgets in the xchat-gnome interface were given a11y metadata in the glade file.

Medium-Difficulty Hackery Love
Fix navigation-pane behavior

The navigation pane behaves strangely in some cases, such as parting/rejoining channels. A lot involved in fixing this is just to determine what the default behaviors ought to be. If you're unsure about your intuitions, come ask us in #xchat-gnome.



irc:// URI support

There are a couple parts to this. The first is a parser for the URI that handles everything in this internet draft. The second is registering URL handlers with gnome.

Difficult Hackery Love
GtkXText a11y support

The XText widget does not include any accessibility support at all. Adding support for the a11y interfaces is probably not a trivial thing, and is probably only a job for someone intimately familiar with how a11y works and willing to deal with a big mess of code in XText.
