xchat-gnome FAQ
General Questions
Q. Who are you people?
A. Overworked, underpaid, disaffected but generally corporeal students.

Q. What is xchat-gnome?
A. xchat-gnome is a new frontend for the popular X-Chat IRC client targetted towards the GNOME platform. Our eventual goals include full desktop integration and HIG compliance.

Q. Why is xchat-gnome?
A. X-Chat has been a popular and successful open-source IRC client for some time now. It is undebateably a stable, powerful and fully featured client, as proven by the number of faithful users it maintains. Unfortunately, X-Chat's user interface became cluttered over time as more features were gradually added. We view xchat-gnome as an opportunity to clean up the X-Chat user interface while (eventually ;)) keeping the stability and power that X-Chat users have come to expect.

Q. Is xchat-gnome a fork?
A. Not really. xchat-gnome is just a new frontend that drops in to a normal xchat source tree. We don't have a separate xchat codebase, and patches are contributed back to xchat where necessary. There's no technical reason why xchat-gnome couldn't become part of the standard xchat distribution, though it's too early to tell if this will ever actually happen.

Q.What happens if xchat goes closed source?
A.Recently, zed has stirred up plenty of emotion by licensing the windows build as shareware. Since this itself is questionable regarding the licensing, it's very unlikely that the linux version will ever be closed as well. However, since people seem to worry about such things, here's the answer: xchat-gnome will always stay GPL.

Q. Ayo Jerks, you got the treeview for windows yet?
A. There are no plans to make xchat-gnome run under windows at this point. If eventually the GNOME libraries are ported, support for windows will probably follow.

Q. You are our slaves! Implement my silly feature requests! Have some drugs!
A. No, and that wasn't a question.
Migrating from X-Chat gtk+
Q. Can xchat-gnome be configured to use tabs?
A. No. Adding this capability would require creating some very messy code, and doesn't really add anything to the program. xchat-gnome is not meant to replace X-Chat gtk+ for power users who have grown attached to its interface, but instead provide an alternative.

Q. Why no tabs?
A. The tree view started as an experiment. It turned out that it is significantly easier to scan visually than a long list of horizontal tabs. It also seems to have the side-effect of removing any guilt that stems from being in a bazillion channels.

Q. You should add tabs
A. No, we shouldn't. And that wasn't a question either.

Q. How is the server list edited?
A. The server list is edited from the preferences window rather than the connect dialog
Compiling and Running
Q. I compiled it and all I got was this lousy 'xchat' binary. Where's xchat-gnome?
A. If you're using 0.1 (which you really shouldn't be at this point), make sure you configured with --enable-gnomefe. You'll find the binary in src/fe-gnome. If you're using 0.2 or the latest SVN, make sure everything was detected properly in the configure stage.
